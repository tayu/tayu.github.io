さて、始めよう

えーと、やっと日記、その他を書く準備ができた。

このページは WiLiKi の記法で 書いている。ただし、マクロは使えない。とにかく、テキストから　html を生成 するための最低限の機能だけ、なんとか使える様になったところ。

将来的に、ページデータを全て dbm に放り込んで全ページを静的に生成 するところまで持って行ければ良いのだが。過去にはあったらしいのだが、今のバージョンでは動かなかった。

作者の人が今、ブログエンジンを作っているらしいけど、こちらは CGI としての実装なので、GitHub には 載らない。

最初は GitHub だし、マークダウンで書こうと思った。 実を言うと、いくつかのフリーのブログでマークダウンを採用するところが 出てきたので、そちらでやりかけてはいた。けど、どうにも 馴染めなかった。で、色々見ていくと、なんだかんだで、どこのサイトも 独自拡張をしているのが見えてきた。GitHub も例外ではなく、 GitHub Flavored Markdown (GFM) として拡張している。

結局のところ、みんな使いにくいのだと思う。

特にダメだったのがコートブロック。もうね、各行の行頭に 4 個のスペースとか、どんだけ面倒な事をさせるのか、と。 各行、で 4 個、と。二重ややこしい。そんなのいちいち数えてらんない。

いや、もちろん、キーボードマクロとか使えば良いんだけど。

一応補足すると、GFM では特別な行で囲む構文を拡張していて、 各行とか行頭とかでとうこうする構文は使わなくとも書ける。 なので、GFM で書く事は最後まで残った選択肢だった。 でも、止めたのは、ローカルのコンバータが無かったから。 ちょっとした修正の確認などで、いちいちコミット＆アップロードするのは どうにも効率が悪すぎる。また、無駄にコミット回数が増えるって事もある。

で、そんなこんなで、マークダウンは諦めて wiki 、それも使い慣れた WiLiKi に する事にした。

したんだけど、コンバータが無い。ただ、 WiLiKi はソースが有ったので、 必要な変換部分だけを切り出せないものか、と考えた。 まぁ、初学者には結構キツかったし、今でもあんまり理解はしていない。 けど、まぁ、何とか切り出せたのでオッケー。時間は掛かったけど。
