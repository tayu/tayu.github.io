最初に

えーと、やっと日記、その他を書く準備ができた。

このページは WiLiKi の記法で 書いている。ただし、マクロは使えない。とにかく、html を生成するための 最低限の機能だけ（または、見つけた分だけとも言う）で変換してるから。

今の所、WiLiKi 記法のテキストファイルを作成し、html にコンバートしている。 コンバート部分だけ、スクリプトを作成した。

WiLiKi本来の動作の通り、dbm から引っ張って一揃いの html を生成する 方が望ましいかもしれない。以前はあったらしいのだが、今のバージョンでは動かなかった。

最初は GitHub だし、マークダウンで書こうと思っていたけど、 どうにもなじめなかった。特にコードブロックで各行の行頭に 4 個 ずつスペースを入れろとか、どんだけ面倒なことをさせるんだって感じで。

確かに、場所によって TAB でも可だったりもする。GitHub は自分の所用に 拡張してたりする。

結局、みんな使いにくいのだろう。独自拡張が多過ぎ。

次善として、GitHub の拡張（GitHub Flavored Markdown）を使おうとも思った。けど、 ローカルで使える GFM から html へのコンバータが見当たらなかった。 一々、修正の確認の為に GitHub サーバへアップロードするのは 作業効率からも、やりたくない。

で、結局、使い慣れてもいる WiLiKi 記法に戻る事にしたというワケ。 ただ、コンバータの用意に手間取った。やっぱり初学者にはあのソースは 難度がちょっと高かった。
